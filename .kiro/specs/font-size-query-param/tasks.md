# Implementation Plan

- [x] 1. ParamParser: フォントサイズパラメータの解析・検証
- [x] 1.1 タイトル・サイト名フォントサイズの解析とバリデーション
  - タイトル用・サイト名用フォントサイズを正の整数として解析し、解析結果に格納する
  - 省略・空文字の場合は「未設定」として扱い、既定値の埋め込みは行わず後段のスケーリング判定に委ねる
  - 正の整数以外（負数・0・小数・非数値文字列）は fail-fast でバリデーションエラーとし、フォントサイズ専用のエラーコードを返す
  - エラーはコード・対象フィールド名・人間可読メッセージを含む既存の構造化エラー形式で返す
  - 既存の正整数検証方針を踏襲し、検証順序はタイトル → サイト名とする
  - 仕様外パラメータを無視する既存挙動を維持する
  - _Requirements: 1.1, 1.2, 2.1, 2.2, 3.1, 3.2, 3.3_

- [x]* 1.2 (P) 解析・検証のユニットテスト
  - 正の整数指定が解析結果に格納され、省略時は「未設定」になることを検証する
  - 0・負数・小数・非数値で専用エラーコードと対象フィールドが返ることを検証する
  - 既存パラメータと併用した際に fail-fast の検証順序が破綻しないことを検証する
  - 1.1 完了が前提（解析結果の型に依存）
  - _Requirements: 1.1, 2.1, 3.1, 3.2, 3.3_

- [x] 2. TemplateRenderer: 条件付きフォントサイズ適用
- [x] 2.1 指定値の優先適用とスケーリング分岐
  - タイトル・サイト名それぞれ、指定値があればスケール係数・最小クランプを適用せず実効フォントサイズに採用する
  - 指定がなければ既定トークンに短辺基準スケール係数と最小クランプ（タイトル 16px・ラベル 12px）を適用する
  - サイト名の指定値を内部のラベルスロットへ対応付ける
  - 確定した実効タイトルサイズを描画とタイトル折り返し計算の双方へ渡す
  - パディングやアクセントライン等、フォントサイズ以外のトークンは従来どおりスケーリングする
  - 1.1 完了が前提（解析結果のフォントサイズフィールドを参照）
  - _Requirements: 1.2, 1.3, 2.2, 2.3, 4.1, 4.2, 4.3, 4.4, 4.5_

- [x] 2.2 タイトル省略計算の境界ガード
  - 実効タイトルサイズがテキスト幅を上回る場合に「1 行あたり文字数」が 0 となる異常を防止する
  - 末尾文字の欠落やクラッシュを起こさず、健全な出力（全文または省略記号付き省略）を返す
  - 2.1 と同一モジュールのため 2.1 の後に実施する
  - _Requirements: 4.5_

- [x]* 2.3 テンプレートのユニットテスト・スナップショット
  - 指定時はスケール・クランプ非適用、省略時は従来式の値になることを検証する
  - オーバーライド省略時に既存スナップショットが不変であること（回帰防止）を検証する
  - タイトルサイズがテキスト幅を上回る境界指定で折り返しが破綻しないことを検証する
  - フォントサイズ指定ケースのスナップショットを新規追加する
  - _Requirements: 1.3, 2.3, 4.1, 4.2, 4.3, 4.4, 4.5_

- [x] 3. 統合検証
  - 型チェックと全ユニット・スナップショットテストを通過させる
  - 指定・省略・不正値の各クエリでエンドポイントの応答（画像生成と 400 エラー）を確認する
  - _Requirements: 1.1, 2.1, 3.1, 4.1, 4.2, 4.3, 4.4_
