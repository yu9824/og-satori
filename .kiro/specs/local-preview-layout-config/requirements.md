# 要件定義書

## はじめに

本ドキュメントは `local-preview-layout-config` フィーチャーの要件を定義する。本フィーチャーは次の 3 つの目的を持つ。

1. **ローカルプレビュー機能**: デプロイ前にブラウザ上で OGP 画像レイアウトを即座に確認する手段を提供する。
2. **設定変更機能**: `AppConfig`（`config.ts` / `.env`）で指定可能なパラメータをプレビュー時にインタラクティブに変更できるようにする。
3. **レイアウト修正**: `textWidth` に基づいてタイトルが中央寄せになるよう修正し、ブログ名とタイトルの左端を揃える。

---

## 要件

### 要件 1: ローカルプレビューページ

**目的:** 開発者として、デプロイ前にブラウザ上で OGP 画像の最終レンダリング結果を確認したい。そうすることで、デプロイ後に意図しないレイアウト崩れを発見するリスクを減らしたい。

#### 受け入れ基準

1. The Preview Tool shall provide a browser-accessible page at a dedicated local route (e.g. `/preview`) that renders the OGP image inline.
2. When the user opens the preview page, the Preview Tool shall display the OGP image generated by `/api/og` with the current configuration defaults.
3. The Preview Tool shall accept all parameters supported by `/api/og` (`title`, `width`, `height`, `textWidth`, `format`) as URL query parameters on the preview page.
4. If the application is running in production (`NODE_ENV=production`), the `/preview/config` page and the `/api/preview-og` endpoint shall each return a 404 response to prevent accidental exposure of configuration parameters. The `/preview` page shall remain accessible in production.

---

### 要件 2: 設定パラメータのインタラクティブ変更

**目的:** 開発者として、プレビューページ上で `AppConfig`（`SITE_NAME`, `DEFAULT_WIDTH`, `DEFAULT_HEIGHT`, `DEFAULT_TEXT_WIDTH_RATIO`）および画像パラメータ（`title`, `width`, `height`, `textWidth`）を変更し、即座に反映結果を確認したい。

#### 受け入れ基準

1. The Preview Tool shall provide UI controls (input fields or sliders) for at least the following values: `title`, `width`, `height`, `textWidth`, `siteName`, `defaultTextWidthRatio`.
2. When any control value changes, the Preview Tool shall re-render the OGP image without requiring a full page reload.
3. The Preview Tool shall initialize each control with the value currently read from environment variables (`.env.local`) or the code defaults defined in `config.ts`.
4. If an invalid value is entered (e.g. non-positive integer for `width`), the Preview Tool shall display a validation error message and shall not re-fetch the image.
5. The Preview Tool shall not persist any user-entered values to disk or environment variables; changes are ephemeral and scoped to the preview session.

---

### 要件 3: `textWidth` に基づく水平中央寄せレイアウト

**目的:** 開発者として、`textWidth` の値に応じてタイトルテキストエリアが画像の水平方向に中央寄せされた状態でプレビューしたい。そうすることで、実際のブログ記事 OGP 画像の見た目を正確に確認できる。

#### 受け入れ基準

1. The OGP Template shall position the title text container so that the left and right margins are equal: `(width − textWidth) / 2` on each side.
2. When `textWidth` equals `width`, the OGP Template shall render the title text container with zero horizontal margin.
3. If `textWidth` is greater than `width`, the OGP Template shall clamp `textWidth` to `width` (treat as `textWidth = width`), ensuring no overflow.
4. The OGP Template shall not apply any fixed left offset to the title text container independently of the `(width − textWidth) / 2` centering formula.

---

### 要件 4: ブログ名とタイトルの左端揃え

**目的:** 開発者として、フッターのブログ名（`siteName`）の左端が、タイトルテキストの左端と水平方向に揃っていることを確認したい。そうすることで、視覚的に整合性のある OGP 画像を生成できる。

#### 受け入れ基準

1. The OGP Template shall align the left edge of the footer area (accent line and `siteName`) with the left edge of the title text container.
2. When `textWidth` is less than `width`, the OGP Template shall apply the same horizontal offset `(width − textWidth) / 2` to both the title text container and the footer area.
3. When `textWidth` equals `width`, the OGP Template shall render the footer area starting at the left edge of the padded content area (zero additional offset).
4. The OGP Template shall not use a separate, independently fixed margin for the footer area; the footer offset shall always be derived from the same centering formula as the title.

---

### 要件 5: フォントサイズ / 解像度スケーリングの設定可能化

**目的:** 開発者として、プレビュー時に画像の幅・高さを変更することで等価的にフォントサイズのスケーリング結果を確認したい。また、`config.ts` に存在するデザイントークン（`TITLE_FONT_SIZE`, `LABEL_FONT_SIZE`）はリポジトリのフォーク時に変更できる形で管理されていることを期待する。

#### 受け入れ基準

1. The OGP Template shall scale `titleFontSize` and `labelFontSize` proportionally to `min(width, height) / BASE_SHORT_SIDE` as currently implemented, so that changing `width` and `height` in the preview controls effectively changes the rendered font size.
2. The Preview Tool shall reflect font size changes immediately when `width` or `height` controls are updated.
3. The OGP Template shall read base font sizes and the scaling reference resolution from the design token constants (`TITLE_FONT_SIZE`, `LABEL_FONT_SIZE`, `BASE_SHORT_SIDE`) defined at the top of `lib/template.tsx`, so that fork users can customize them without touching other logic.
4. The Preview Tool (`/preview/config`) shall provide a UI control for `BASE_SHORT_SIDE`, allowing the developer to preview the effect of different scaling reference resolutions on the rendered font size without modifying source code. The value entered via this control is ephemeral and scoped to the preview session.
